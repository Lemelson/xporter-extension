XPorter Privacy Policy

Chrome Extension for Exporting X (Twitter) Data

Overview

Your X (Twitter) data never leaves your browser — XPorter has no servers and no account. The only thing it ever sends is an anonymous, non-personal usage summary when you uninstall, used solely to understand why people leave and to improve the extension. No personal data and none of your exported content is included.

What Data the Extension Accesses

XPorter accesses the following data to perform exports that you initiate and to build the local seen-post dataset described below:

  • X.com session cookies — used to authenticate API requests on your behalf. The extension reads your existing login session to make the same API calls that X.com itself makes in your browser. No credentials are stored or copied.
  • X.com API responses — profile information, posts, follower/following lists retrieved via X.com's public-facing APIs. This data is processed locally to generate export files (CSV, JSON, or XLSX).
  • Posts already loaded while you browse X.com — non-reply posts and their public engagement/profile metrics are deduplicated by post ID and stored locally in IndexedDB. XPorter makes no additional request for this collection and does not store the page URL.
  • Extension settings — your preferences (language, theme) are stored locally using Chrome's storage.local API.

What the Extension Does NOT Do

  • Does not collect or store your X.com password
  • Does not send your exported data, X content, usernames, or browsing history anywhere
  • Does not use any analytics, advertising, or tracking SDKs during normal use
  • Does not store your browsing history or page URLs; the local dataset contains post data and the X timeline operation that supplied it
  • Does not modify your X.com account in any way (no posting, following, or liking)
  • Does not inject ads or promotional content
  • Does not use any remote code — all code runs locally within the extension package

Anonymous Uninstall Feedback

When you uninstall XPorter, Chrome opens a short feedback page. To understand why people leave and what to fix, XPorter attaches an anonymous, non-personal usage summary to that page. It contains only aggregate product stats:

  • Extension version, your operating system (e.g. Windows / macOS / Linux), and how long it was installed (install date and the time elapsed until uninstall)
  • Interface language and theme (light/dark)
  • How many times you opened the extension and roughly how much time you actively spent in it
  • How many exports you ran, how many succeeded or failed, and a short error code for the last failure
  • How much time had passed since your last export
  • Counts per export type (posts, followers, following, verified followers) and per format (CSV / JSON / XLSX)
  • The approximate total number of items exported and a few non-identifying settings (e.g. include retweets/replies, quantity limit)
  • How long the feedback page itself was open

This summary contains no usernames, no exported content, no X data, and nothing that identifies you. If you choose to leave a written comment on the feedback page, that text is sent too — only what you type. The data is stored in a private spreadsheet via Google Apps Script / Google Sheets (Google acts as the data processor). It is used only to improve XPorter and is never sold or shared.

This is the only data XPorter ever transmits anywhere other than X.com. The anonymous summary is recorded when the feedback page loads; any written comment is sent only if you choose to submit one.

Permissions Explained

cookies

Required to read authentication tokens (CSRF token) from your X.com session to make authenticated API requests.

activeTab

Used to detect when you are on an X.com profile page and to identify the username of the profile being viewed.

downloads

Used to save exported CSV, JSON, or XLSX files to your Downloads folder when you click the download button.

storage and unlimitedStorage

Used to save extension preferences, resumable export batches, recent export history, and anonymous usage counters locally in Chrome, plus the deduplicated seen-post dataset in IndexedDB. unlimitedStorage prevents larger local datasets and exports from being truncated by Chrome's normal storage quota.

host_permissions for x.com

Required to make API calls to X.com's servers to retrieve the data you've requested to export.

Data Storage

All data is stored locally in your browser:

  • Export rows are saved in batches to Chrome's storage.local so an interrupted export can be resumed without losing already saved rows
  • Once downloaded as a CSV, JSON, or XLSX file, the data is saved to your local filesystem (Downloads folder)
  • The current export and data snapshots for up to five recent history entries may remain locally until they expire, you start or clear an export, clear history, or uninstall the extension
  • The seen-post dataset keeps one row per post ID, retains the 50,000 most recently seen unique posts, and remains until you clear it in Settings or uninstall the extension
  • Extension settings are stored in Chrome's storage.local and can be cleared by uninstalling the extension
  • A small anonymous usage counter (including opens, active time, export counts, install date, and last error code) is kept in storage.local to build the uninstall summary, and is removed when you uninstall

Third-Party Services

XPorter does not integrate with any analytics platforms, advertising networks, or trackers. Apart from X.com (used to fetch the data you export), the only external communication is the anonymous uninstall feedback described above, which is stored via Google (Apps Script / Google Sheets). XPorter loads no remote code.

Data Security

All communication with X.com's servers uses HTTPS encryption. Authentication tokens are handled in memory and are never written to disk or transmitted to any server other than X.com.

Children's Privacy

XPorter is not directed at children under 13 and does not knowingly collect information from minors.

Changes to This Policy

If this privacy policy is updated, the changes will be reflected on this page with an updated effective date. Significant changes will be noted in the extension's changelog.
