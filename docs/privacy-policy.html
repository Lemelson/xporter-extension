Privacy Policy Plain English

Your X (Twitter) data never leaves your browser — XPorter has no servers and no account. The only thing it ever sends is an anonymous, non-personal usage summary when you uninstall, used solely to understand why people leave and to improve the extension. No personal data and none of your exported content is included.

What the extension accesses

XPorter accesses the following only to perform the exports you start:

  • X.com session cookies — to authenticate API requests on your behalf, making the same calls X.com already makes in your browser. No credentials are stored or copied.
  • X.com API responses — profile info, posts and follower/following lists, processed locally to build your export file (CSV, JSON or XLSX).
  • Extension settings — your preferences (language, theme, export options) stored locally via Chrome's storage.local.

What XPorter does NOT do

  • Does not collect or store your X.com password
  • Does not send your exported data, X content, usernames, or browsing history anywhere
  • Does not use analytics, advertising, or tracking SDKs during normal use
  • Does not modify your X.com account (no posting, following or liking)
  • Does not inject ads or promotional content
  • Does not load any remote code — everything runs from the extension package

Anonymous uninstall feedback

When you uninstall XPorter, Chrome opens a short feedback page. To understand why people leave and what to fix, XPorter attaches an anonymous, non-personal usage summary to that page. It contains only aggregate product stats:

  • Extension version and how many days it was installed
  • Interface language and theme (light/dark)
  • How many exports you ran, how many succeeded or failed, and a short error code for the last failure
  • Counts per export type (posts, followers, following, verified followers) and per format (CSV / JSON / XLSX)
  • The approximate total number of items exported and a few non-identifying settings (e.g. include retweets/replies, quantity limit)

This summary contains no usernames, no exported content, no X data, and nothing that identifies you. If you choose to leave a written comment on the feedback page, only the text you type is sent.

The data is stored in a private spreadsheet via Google Apps Script / Google Sheets (Google acts as the data processor). It is used only to improve XPorter and is never sold or shared. This is the only data XPorter ever transmits anywhere other than X.com. The anonymous summary is recorded when the feedback page loads; any written comment is sent only if you choose to submit one.

Permissions explained

cookies

Reads authentication tokens (CSRF token) from your X.com session to make authenticated API requests.

activeTab & tabs

Detects when you're on an X.com profile and identifies the username being viewed.

downloads

Saves the exported CSV, JSON or XLSX file to your Downloads folder when you click download.

storage

Stores your preferences and a small anonymous usage counter locally in Chrome's storage.

Host access to x.com

Required to make API calls to X.com to retrieve the data you asked to export.

Data storage

  • Export data is held in memory during the export and saved to your Downloads folder when you download it
  • Settings are stored in Chrome's storage.local and cleared when you uninstall
  • A small anonymous usage counter (export counts, install date, last error code) is kept in storage.local to build the uninstall summary, and removed when you uninstall

Third-party services

XPorter integrates with no analytics platforms, ad networks or trackers. Apart from X.com (used to fetch the data you export), the only external communication is the anonymous uninstall feedback above, stored via Google (Apps Script / Google Sheets). XPorter loads no remote code.

Data security

All communication with X.com uses HTTPS. Authentication tokens are handled in memory and are never written to disk or sent to any server other than X.com.

Children's privacy

XPorter is not directed at children under 13 and does not knowingly collect information from minors.

Changes to this policy

Updates will be reflected on this page with a new effective date. Significant changes will be noted in the extension's changelog.
